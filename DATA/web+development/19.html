About the Job

Entry-level IT Position
Help Desk and Web Developer

(Trainer and Help Desk Specialist I)

The Office of the Revisor of Statutes, a nonpartisan office in the Minnesota Legislature, is seeking applicants for a Trainer and Help Desk Specialist I. The minimum qualification is a 2-year associate degree in computer science; a bachelor degree is preferred. Work experience of 1 or more years is desirable. The position has two primary responsibilities, split approximately 50/50% in terms of time, Help Desk and Software Development of web programs.  Knowledge of desktop hardware and MS Office are required. Salary is dependent upon experience. The starting salary is $46,868. This is a full-time regular position with benefits, beginning immediately. See the full job descriptions for details.

This position involves various activities each day. Activities range from sedentary office work to physically moving equipment and visiting users in separate buildings. It requires working individually, in teams, and delivering training to users. Good communication, problem-solving, and organization skills are required, along with a desire to work problems to completion and continuously learn.

Help Desk duties are delivering first-level support to users of Revisor information systems. Support is provided to all legislative offices and the public. Duties include:

         Answer the Help Desk phone and emails.

         Carry the after-hours Help Desk pager and cell phone, on a rotating schedule, during the legislative session and as needed.

         Log all user issues and generate reports for the IS team

         Serve as the focal point for communications between users and the IS staff.

         Responsible for the desktop hardware used by 50 Revisor staff. This hardware includes: VOIP telephones, Wyse thin clients, computer monitors, network laser printers.

         Develop and deliver training on software applications. These applications include: a custom built document publishing system, the Revisor web site and intranet, email, and various commercial applications including Microsoft Office.

         Create documentation for users.

Software Development duties are web programming.  The Revisor web server runs 3 different types of programs: a) public web pages; b) intranet pages for legislative staff only; c) automated programs for updating the Revisor database with external data.  Programming languages used are: PHP, JavaScript, SQL and XML.  The offices' lead web programmer will monitor assignments and provide assistance as needed.  Duties include:

 

1.      Being the backup to the lead web developer.  Have a working knowledge of all 3 types of programs (a-c above).

2.      Convert existing Intranet pages to a new page layout, based on the 2012 LNET standard.  Complete by December 31, 2012.

3.      Maintain the 12 existing intranet applications.  Fix known problems with the existing pages.

4.      Finish building, then maintain a fully automated testing system of all: a) public; and b) intranet web pages.  When completed, the automated testing system will run nightly and produce reports showing:

a.       Broken pages

b.      Pages with poor performance

5.      Develop new intranet pages, as needed.  These may include:

a.       A purchase order creation system

b.      A budget tracking system that maps each purchase order line item to a SWIFT accounting code.

c.       An inventory tracking system

d.      Generate and print reports from the above systems data.

6.      Resolve printing issues in XTEND and MS Windows operating system.

a.       Resolve issue reported during 2012 session - slow PDF printing.

b.      Resolve issue - slow MSWord printing when document contains color graphic.             

c.       Learn and troubleshoot XTEND code for printing.

d.      Learn and troubleshoot Microsoft-OS, print-server operation.

7.      Perform basic project management tasks for software development including:

a.       User needs meetings.

b.      Requirements collection and documentation

c.       Task and schedule documentation

d.      Software design documents
